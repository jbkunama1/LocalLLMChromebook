Setup in drei Blöcken

llama.cpp wird gebaut, ein GGUF-Modell lokal bereitgestellt und anschließend per llama-server veröffentlicht. Cloudflare Tunnel kann den Port 8080 sicher nach außen bereitstellen. [web:110][web:121][web:9]

1 · Build

llama.cpp kompilieren

CMake-Build direkt im Linux-Container.

git clone https://github.com/ggml-org/llama.cpp
cd llama.cpp
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
2 · Modell

GGUF bereitstellen

Kleines quantisiertes Modell lokal nach ~/models legen.

mkdir -p ~/models
# Datei nach ~/models/dein-modell.gguf
# z.B. TinyLlama oder SmolLM2 als GGUF
3 · Server

llama-server starten

Server lokal starten und dann per Cloudflare tunneln.

./build/bin/llama-server -m ~/models/dein-modell.gguf \
  --host 0.0.0.0 --port 8080 -c 1024 -ngl 0

FAQ

llama.cpp arbeitet direkt mit GGUF-Dateien und vermeidet die zusätzliche Modellverwaltung von Ollama. Das ist oft robuster, wenn nur ein kleines, lokal kontrolliertes Setup gewünscht ist. [web:117]
Du brauchst ein lokales GGUF-Modell im Dateisystem und startest es direkt mit llama-server. Der Server ist Bestandteil von llama.cpp. [web:117][web:121]
Im Cloudflare-Dashboard wird ein Tunnel erstellt und ein Public Hostname auf localhost:8080 gesetzt. [web:9]
