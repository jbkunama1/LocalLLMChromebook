Setup in drei Blöcken

Die vollständige Anleitung findest du im README im GitHub‑Repo. Hier die Kurzfassung der wichtigsten Befehle.

1 · Backend (Ollama)

Ollama installieren & Modell laden

Installiere Ollama, lade ein leichtes, aber sinnvolles Modell wie de>qwen2.5:3b und teste lokale Antworten direkt im Terminal. 

de>curl -fsSL https://ollama.com/install.sh | sh
ollama pull qwen2.5:3b
ollama run qwen2.5:3b "Hallo Chromebook!"
Empfohlen: ≥ 4 GB RAM
2 · Chat UI (Open WebUI)

Open WebUI als lokale Chat‑Oberfläche

Installiere Open WebUI via de>pip, starte die Weboberfläche und verknüpfe sie mit deinem Ollama‑Backend unter de>http://localhost:11434. 

de>python3 -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install open-webui
open-webui serve
UI: http://localhost:8080
3 · Internetzugriff (Cloudflare)

Sicherer Zugriff mit Cloudflare Tunnel

Richte über das Cloudflare‑Dashboard einen Tunnel ein, der die Ollama‑API und optional Open WebUI über eigene Subdomains erreichbar macht – ohne Port‑Forwarding. 

de># Befehl aus dem Cloudflare-Dashboard
sudo cloudflared service install <DEIN-TOKEN>
# Hostnames:
# ollama.deinedomain.de → http://localhost:11434
# chat.deinedomain.de   → http://localhost:8080

FAQ · Häufige Fragen

Kurz beantwortet, was beim ersten Setup typischerweise aufkommt.
Brauche ich zwingend Docker für Open WebUI? +
Nein. Für Chromebooks ist die Installation per de>pip oft leichter und spart Ressourcen. Docker geht auch, benötigt aber mehr RAM und Speicherplatz. [web:23][web:19]
Welches Modell eignet sich für 4 GB RAM? +
Modelle wie de>qwen2.5:3b oder de>gemma3:2b sind ein guter Start: vernünftige Qualität bei moderatem RAM‑Verbrauch und 3–5 Tokens/Sekunde je nach CPU. [web:10][web:7]
Ist der Zugriff über Cloudflare Tunnel sicher? +
Ja, besonders wenn du zusätzlich Cloudflare Access aktivierst (z.B. E‑Mail‑Login oder SSO). So kann nicht jeder beliebige Nutzer auf deine Instanz zugreifen. [web:9]
