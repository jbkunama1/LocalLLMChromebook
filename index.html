Lokale GGUF-Modelle mit llama.cpp.

Dieses Projekt nutzt llama.cpp direkt als Inference-Engine, startet den eingebauten HTTP-Server lokal auf dem Chromebook und schützt Zugriffe optional per API-Key-Datei. Die Server-Doku nennt dafür `--api-key-file`, und Cloudflare Tunnel kann lokale Dienste auf `localhost:8080` per Public Hostname veröffentlichen. [web:121][web:218]

Setup in drei Blöcken

Der Standard-Build von llama.cpp läuft über CMake, `llama-server` unterstützt Host, Port und API-Key-Datei, und Cloudflare Tunnel veröffentlicht lokale Dienste über Public Hostnames wie `app.example.com → http://localhost:8080`. [web:121][web:219][web:223]

mkdir -p ~/models

Mit API-Key starten

./start-llama.sh

curl http://localhost:8080/health \
  -H "Authorization: Bearer sk-xxxxxxxx"

API-Key

`llama-server` kann eine API-Key-Datei einlesen; wenn diese gesetzt ist, müssen Requests einen passenden Bearer-Key mitsenden. [web:121][web:124]

API_KEY_FILE="$HOME/.config/llama.cpp/api-keys.txt"
printf '%s\n' 'sk-xxxxxxxx' > "$API_KEY_FILE"
chmod 600 "$API_KEY_FILE"

curl http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-xxxxxxxx" \
  -d '{
    "model": "gpt-3.5-turbo",
    "messages": [{"role":"user","content":"Hallo Chromebook"}]
  }'
